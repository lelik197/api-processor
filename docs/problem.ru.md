# Описание проблемы

Основная задача, которую решает это приложение — **Анализ прибыльности мобильных приложений**.
Разработка мобильного приложения сопряжена со сложным набором переменных затрат: от закупки устройств и серверной инфраструктуры до зарплат смешанных команд (разработчики, QA). Без единого вычислительного ядра трудно точно предсказать, будет ли проект прибыльным, или определить точку безубыточности.

## Ключевые функции и сценарии использования

1.  **Расчет структуры затрат и прибыльности**
    *   Агрегирует все драйверы затрат: оборудование (устройства, серверы) и человеческий капитал (зарплаты разработчиков/QA).
    *   Предоставляет точную базовую оценку "Инвестиционных затрат", которая является знаменателем для любой формулы ROI (окупаемости инвестиций) или расчета прибыли.

2.  **Сравнительный финансовый анализ**
    *   Позволяет заинтересованным сторонам сравнивать различные финансовые сценарии.
    *   Пример: Сравнение затрат на инфраструктуру и затрат на персонал (например, `server_price` против `qa_salary`) или сравнение стоимости двух разных стратегий развертывания для максимизации маржи.

3.  **Анализ чувствительности и оценка рисков**
    *   Анализирует, как волатильность конкретных факторов затрат (например, рост `server_price`) влияет на общую финансовую модель.
    *   Помогает в стресс-тестировании бизнес-модели против возможных изменений рынка.

## Техническое решение

Сервис предоставляет REST API (`/api/calc`), выступающий в роли "финансового мозга" для мобильного проекта.

### Поддерживаемые операции
*   `calculate`: Вычисляет совокупную стоимость владения (TCO) на основе количества ресурсов и цен за единицу.
*   `compare`: Оценивает две финансовые метрики (например, `server_price`, `qa_salary`) для определения оптимального распределения средств.
*   `sensitivity`: Прогнозирует влияние колебаний затрат на заданную переменную (например, `server_price`).
